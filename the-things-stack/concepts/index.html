Basic Concepts

This section introduces basic concepts of working with The Things Stack.

We start with a general technical discussion of The Things Stack architecture and progress to other more detailed topics that can be found in the left menu.

Architecture

The Things Stack follows a API driven microservice architecture, well suited for high availability and reliability. The Things Stack has the following main components. Component (Service) Short Description Gateway Server GS Maintains connections with gateways supporting the Basic Station, UDP, MQTT and gRPC protocols. It forwards uplink traffic to Network Servers directly or indirectly, and schedules downlink traffic on gateways. Network Server NS Handles the LoRaWAN® network layer, including MAC commands, regional parameters and adaptive data rate (ADR).

Networking

The Things Stack uses a port per protocol, with a TLS counterpart when applicable. Ports can be shared by multiple services using the same protocol, i.e. gRPC services sharing management, data and events services.
