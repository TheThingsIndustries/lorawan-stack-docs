Command-Line Interface

This is the reference for the command-line interface of The Things Stack: ttn-lw-cli.

Basic Commands

Command ttn-lw-cli

Description

The Things Industries Command-line Interface

Usage

ttn-lw-cli [flags]

Flags for ttn-lw-cli

The following flags are also available in sub-commands:

--allow-unknown-hosts

type: bool

default: false

Allow sending credentials to unknown hosts

--application-server-enabled

type: bool

default: true

Application Server enabled

--application-server-grpc-address

type: string

default: localhost:8884

Application Server address

--ca

type: string

CA certificate file

-c
--config

type: stringSlice

default: [.ttn-lw-cli.yml,$HOME/.ttn-lw-cli.yml,$HOME/Library/Application Support/.ttn-lw-cli.yml]

Location of the config files

--credentials-id

type: string

Credentials ID (if using multiple configurations)

--device-claiming-server-grpc-address

type: string

default: localhost:8884

Device Claiming Server address

--device-template-converter-grpc-address

type: string

default: localhost:8884

Device Template Converter address

--dump-requests

type: bool

default: false

When log level is set to debug, also dump request payload as JSON

--gateway-server-enabled

type: bool

default: true

Gateway Server enabled

--gateway-server-grpc-address

type: string

default: localhost:8884

Gateway Server address

--identity-server-grpc-address

type: string

default: localhost:8884

Identity Server address

--input-format

type: string

default: json

Input format

--insecure

type: bool

default: false

Connect without TLS

--join-server-enabled

type: bool

default: true

Join Server enabled

--join-server-grpc-address

type: string

default: localhost:8884

Join Server address

--log.level

type: string

default: info

The minimum level log messages must have to be shown

--network-server-enabled

type: bool

default: true

Network Server enabled

--network-server-grpc-address

type: string

default: localhost:8884

Network Server address

--oauth-server-address

type: string

default: https://localhost/oauth

OAuth Server address

--output-format

type: string

default: json

Output format

--packet-broker-agent-grpc-address

type: string

default: localhost:8884

Packet Broker Agent address

--qr-code-generator-grpc-address

type: string

default: localhost:8884

QR Code Generator address

Command ttn-lw-cli use

Description

Use

Aliases

generate-configuration generate-cfg

Usage

ttn-lw-cli use [host] [flags]

Flags for ttn-lw-cli use

--fetch-ca

type: bool

default: false

Connect to server and retrieve CA

--grpc-port

type: int

default: 0

--host

type: string

default: localhost

Server host name

--overwrite

type: bool

default: false

Overwrite existing config files

--user

type: bool

default: false

Write config file in user config directory

This command inherits flags from ttn-lw-cli

Command ttn-lw-cli login

Description

Login

Usage

ttn-lw-cli login [flags]

Flags for ttn-lw-cli login

--api-key

type: string

API key to login with (instead of using OAuth)

--callback

type: bool

default: true

use local OAuth callback endpoint

This command inherits flags from ttn-lw-cli

Command ttn-lw-cli logout

Description

Logout

Usage

ttn-lw-cli logout [flags]

Flags for ttn-lw-cli logout

This command inherits flags from ttn-lw-cli

Command ttn-lw-cli config

Description

View the current configuration

Usage

ttn-lw-cli config [flags]

Flags for ttn-lw-cli config

--env

type: bool

default: false

print as environment

--yml

type: bool

default: false

print as yml

This command inherits flags from ttn-lw-cli
