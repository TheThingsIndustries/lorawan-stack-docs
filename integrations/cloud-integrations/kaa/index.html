Kaa

This section shows how to implement an integration between The Things Stack and Kaa IoT platform in order to create a digital twin of your LoRa device and observe your device’s telemetry in the Kaa web interface.

Prerequisites

  1. A user account in the Kaa cloud.

Setup Kaa

First, create an application in the Kaa cloud that will correspond your The Things Stack application. Let’s name it smart-home-application.

Next, create the application version by clicking on the + button as shown on the image below.

Navigate to MQTT under Integrations on the left hand menu in The Things Stack to obtain MQTT connection details needed to proceed with building the integration.

Now you can configure the integration in Kaa. You can add it if you navigate to Device management on the left hand menu, then to Integrations. Fill in the following fields when registering the integration:

  • TTN Username - the MQTT Username from your The Things Stack application
  • TTN API key - the MQTT Password from your The Things Stack application
  • Identity Server Host - The Things Stack Identity Server host address (see the Server Addresses section)
  • MQTT Server Host - the MQTT Public address from your The Things Stack application

Press the Create button to finish creating the integration.

After application integration is created, it is possible to create device integrations, i.e. mappings between The Things Stack devices and Kaa endpoints.

Go to the created application integration and create a device integration by choosing the application version and The Things Stack device that you want to integrate.

On the image below, you can see that the device integration has been created and it appeared in the Devices list.

By navigating to Device management → Devices, you will find the endpoint that was automatically created during the integration creation. All incoming data from The Things Stack device will from now on be ingested under that endpoint.

Visualize Your Data

Next, we want visualize some data from The Things Stack on Kaa UI. To do so, the Kaa application must first be prepared first.

Edit the application configuration for the Endpoint Time Series service (EPTS). EPTS is a Kaa platform component responsible for transforming raw data samples into well-structured time series. It also stores the time series data and provides access to API for other services, including the Web Dashboard.

Enable the time series auto-extraction from data samples for the previously specified Kaa application.

With this function enabled, Kaa will automatically create a time series for each numeric field that it encounters at the root of data samples submitted by your endpoints. You will then be able to view these time series in the Kaa UI, with no extra configuration required.

Now navigate to the device details page of the endpoint created in previous steps. To do that, click on the corresponding row in the device table on the Devices dashboard. The data coming from The Things Stack device will be available on the Device telemetry widget.

Let’s assume that The Things Stack device is sending temperature measurements. You will need to use a payload formatter in order to get the data into a format recognizable by the Kaa platform, such as:

{
    "temperature":20
}

When some uplinks are sent from The Things Stack device, the visualized data on Kaa will look like on the image below.

Congratulations, you have successfully integrated The Things Stack with Kaa and visualized your data in the Kaa UI!
